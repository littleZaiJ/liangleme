# 凉了么 (Liang Le Me) - MVP 产品需求文档

## 1. 产品概述
* **产品名：** 凉了么 (Liang Le Me)
* **核心价值：** 社交止损工具。通过记录对方回复的等待时间，量化用户的“卑微程度”，并提供毒舌的情感分析。
* **Slogan：** 别等了，再等就馊了。
* **风格基调：** 极简主义、黑色幽默、克莱因蓝（忧郁）+ 纯黑。

## 2. 功能架构 (MVP)
App 仅包含三个底部 Tab 页面：
1.  **停尸房 (Timer)**：主页，核心计时功能。
2.  **病历单 (Stats)**：数据统计页。
3.  **尸检科 (Analysis)**：聊天记录分析页。

## 3. 详细功能说明

### 3.1 Tab 1: 停尸房 (Timer) - 核心页
* **交互逻辑：**
    * **初始状态：** 屏幕中央一个巨大的圆形按钮，文案：“消息已发，开始卑微”。
    * **计时状态：** 点击后按钮变为“对方已回，结束卑微”。屏幕显示巨大的计时数字（HH:MM:SS）。
    * **结束状态：** 再次点击按钮，弹窗询问：“他/她回消息了吗？”，确认后停止计时，数据存入本地数据库。
* **视觉反馈 (核心亮点)：**
    * 背景颜色随时间渐变：
        * 0 - 10分钟：**微粉色** (期待)
        * 10分钟 - 2小时：**克莱因蓝** (冷淡)
        * 2小时 - 24小时：**深灰色** (凉了)
        * 24小时+：**纯黑色** (彻底死透)
* **随机毒舌弹幕：** 计时过程中，屏幕上方随机飘过文案（例：“他在回了，在回别人了”）。

### 3.2 Tab 2: 病历单 (Stats) - 统计页
* **数据展示：**
    * **累计等待时长：** 总共浪费了多少人生。
    * **平均响应时间：** 对方平均多久回一次。
    * **卑微指数 (0-100)：** 算法 = (平均等待时长 / 10分钟) * 10。超过 90 分显示红色警报。
* **列表记录：** 按时间倒序展示每一次的等待记录（显示日期、时长、状态）。

### 3.3 Tab 3: 尸检科 (Analysis) - 分析页
* **输入方式：** * MVP阶段支持**文本粘贴**（将聊天记录复制进来）。
    * (预留图片上传入口，MVP暂不实现OCR，仅做UI占位)。
* **分析逻辑 (本地模拟 AI)：**
    * 检测关键词（如：哈哈、嗯、哦、洗澡、去忙）。
    * 检测字数差（我发的字数 vs 对方发的字数）。
* **输出报告卡片：**
    * **死因：** 如“敷衍性回复”、“已读乱回”。
    * **建议：** 如“建议拉黑”、“在那边多烧点纸”。

## 4. 非功能需求
* **数据存储：** 使用 SwiftData 或 CoreData 本地存储，无需联网。
* **UI模式：** 强制深色模式 (Dark Mode)。
* **字体：** 使用衬线体 (Serif) 增加严肃荒诞感。
